# HERMES

A concise and versatile group intelligence engine that predicts everything.

**Rehearse the future in a digital sandbox, and win decisions after countless simulations.**

## Overview

**Hermes** is a multi-agent social simulation and prediction engine. It transforms unstructured documents -- news articles, policy drafts, financial reports, or creative writing -- into a parallel digital world where hundreds of AI-powered agents with distinct personalities freely interact on simulated social media platforms (Twitter and Reddit). The system then produces analytical reports about the emergent behavior, letting you see how scenarios might unfold before they happen.

> **You provide:** Seed documents + a natural language description of what you want to predict.
>
> **Hermes returns:** A detailed prediction report and an interactive digital world you can interrogate.

### Use Cases

- **Policy Analysis**: Test how a new policy might be received before implementing it
- **Public Opinion Prediction**: Simulate how news events might spread and be discussed
- **Creative Exploration**: Explore "what if" scenarios for novels, worldbuilding, or fiction
- **Strategic Planning**: Rehearse PR strategies, product launches, or crisis responses at zero risk
- **Financial Forecasting**: Model market reactions to events using agent-based simulation

## How It Works

Hermes runs a 5-stage pipeline that goes from raw text to interactive prediction:

```
Seed Documents в”Җв”Җв–ә Knowledge Graph в”Җв”Җв–ә Agent Personas в”Җв”Җв–ә Simulation в”Җв”Җв–ә Report в”Җв”Җв–ә Chat
```

### Stage 1: Graph Building (Knowledge Graph Construction)

1. Upload documents (PDF, Markdown, TXT) and describe your simulation requirement
2. Text is extracted, chunked (500 chars with 50-char overlap, respecting sentence boundaries)
3. An LLM analyzes the content and generates an **ontology** -- a schema defining entity types (Student, Professor, MediaOutlet, etc.) and relationship types (WORKS_FOR, STUDIES_AT, etc.)
4. Text chunks are sent to **Zep Cloud** as episodes, which automatically extracts entities and relationships using its built-in NLP pipeline
5. The ontology is registered with Zep to guide extraction toward meaningful, custom-typed nodes

### Stage 2: Environment Setup (Agent Generation)

1. All entities are read from the Zep graph, filtered to custom-labeled nodes (not generic "Entity" labels)
2. For each entity, a detailed **OASIS Agent Profile** is generated via LLM:
   - Bio, background, personality (MBTI), age, gender, profession
   - Social media behavior patterns, interests, stance, unique traits
   - Different prompts for individuals vs. organizations
3. A **simulation configuration** is auto-generated by LLM:
   - **Time config**: Total simulated hours, minutes per round, peak/off-peak hours with activity multipliers
   - **Event config**: Hot topics, narrative direction, initial seed posts
   - **Per-agent config**: Activity level, posts/hour, comments/hour, active hours, sentiment bias, stance, influence weight

### Stage 3: Simulation (Multi-Agent Execution)

1. The **OASIS** (Open Agent Social Interaction Simulations) engine runs as a subprocess
2. Agents interact on **Twitter** and **Reddit** platforms in parallel:
   - **Twitter actions**: CREATE_POST, LIKE_POST, REPOST, FOLLOW, QUOTE_POST, DO_NOTHING
   - **Reddit actions**: CREATE_POST, CREATE_COMMENT, LIKE_POST, DISLIKE_POST, LIKE_COMMENT, DISLIKE_COMMENT, SEARCH_POSTS, SEARCH_USER, TREND, REFRESH, FOLLOW, MUTE, DO_NOTHING
3. Each round represents a time slice (~60 simulated minutes); agents decide actions based on persona, current state, and activity config
4. Platform algorithms determine content visibility using recency and popularity weights
5. Real-time monitoring tracks round progress, action counts, and recent agent activities
6. Optional: Agent activities stream back to Zep graph as new episodes during simulation

### Stage 4: Report Generation (ReACT Agent)

A **Report Agent** (ReACT-style LLM agent) generates a "Future Prediction Report":

1. **Planning**: LLM creates an outline with 2-5 sections based on simulation results
2. **Generation**: For each section, runs a ReACT loop:
   - **Thought**: What information do I need?
   - **Action**: Call a tool (InsightForge, PanoramaSearch, QuickSearch, InterviewAgents)
   - **Observation**: Analyze tool results
   - **Final Answer**: Write section content in markdown
3. Sections are generated sequentially and displayed in real-time as they complete

**Report Agent Tools:**
- **InsightForge**: Decomposes queries into sub-questions, searches each, extracts related entities, builds relationship chains
- **PanoramaSearch**: Full graph view including historical/expired facts (temporal graph features)
- **QuickSearch**: Simple semantic search with local keyword fallback
- **InterviewAgents**: Selects relevant agents, generates interview questions, calls OASIS interview API

### Stage 5: Deep Interaction

After the report is complete, you can:
- **Chat with the Report Agent** -- ask follow-up questions; the agent searches the graph for answers
- **Interview individual agents** -- send questions to specific simulated agents and get in-character responses
- **Batch interview** -- ask multiple agents the same question simultaneously
- **Browse** the full report, agent profiles, and simulation timeline

### Technology Stack

| Component | Technology |
|-----------|------------|
| Backend | Python / Flask (port 5001) |
| Frontend | Vue 3 + Vite (port 3000) |
| Knowledge Graph | Zep Cloud API (GraphRAG) |
| LLM | OpenAI-compatible API (OpenRouter, Qwen, etc.) |
| Simulation Engine | OASIS (external Python library, run as subprocess) |
| Deployment | Docker (single container) |
| i18n | 7 languages: Chinese, English, Spanish, French, Portuguese, Russian, German |

## Quick Start

### Prerequisites

| Tool | Version | Description |
|------|---------|-------------|
| **Node.js** | 18+ | Frontend runtime, includes npm |
| **Python** | >=3.11, <=3.12 | Backend runtime |
| **uv** | Latest | Python package manager |

### 1. Configure Environment Variables

```bash
cp .env.example .env
```

Edit `.env` with your API keys:

```env
# LLM API (planning, simulation config, reports, chat)
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://openrouter.ai/api/v1
LLM_MODEL_NAME=qwen/qwen3.6-plus:free

# Zep Cloud (knowledge graph + GraphRAG)
# Free monthly quota: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

### 2. Install Dependencies

```bash
# One-click: all dependencies (root + frontend + backend)
npm run setup:all
```

Or step by step:

```bash
npm run setup              # Node dependencies (root + frontend)
npm run setup:backend      # Python dependencies (auto-creates venv)
```

### 3. Start Services

```bash
npm run dev                # Start both frontend and backend
```

| Service | URL |
|---------|-----|
| Frontend | http://localhost:3000 |
| Backend API | http://localhost:5001 |

```bash
npm run backend            # Start backend only
npm run frontend           # Start frontend only
```

### Docker Deployment

```bash
cp .env.example .env
docker compose up -d
```

Reads `.env` from root directory, maps ports `3000` (frontend) and `5001` (backend), persists `./backend/uploads`.

## API Reference

### Graph & Project Management (`/api/graph/`)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/ontology/generate` | Upload files + requirement, generate ontology |
| POST | `/build` | Build knowledge graph from project |
| GET | `/project/<id>` | Get project details |
| GET | `/project/list` | List all projects |
| DELETE | `/project/<id>` | Delete project |
| GET | `/task/<id>` | Get async task status |
| GET | `/data/<graph_id>` | Get full graph data (nodes + edges) |

### Simulation Management (`/api/simulation/`)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/create` | Create new simulation instance |
| POST | `/prepare` | Prepare simulation (read entities, generate profiles, generate config) |
| POST | `/start` | Start OASIS simulation |
| POST | `/stop` | Stop running simulation |
| GET | `/<id>/run-status` | Get real-time run status |
| GET | `/<id>/run-status/detail` | Get detailed status with recent actions |
| GET | `/<id>/actions` | Get action history (paginated) |
| GET | `/<id>/agent-stats` | Get per-agent statistics |
| GET | `/<id>/timeline` | Get timeline by round |
| POST | `/interview/batch` | Batch interview agents |
| GET | `/<id>/profiles` | Get agent profiles |

### Report Generation (`/api/report/`)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/generate` | Generate report (async) |
| GET | `/<id>/sections` | Get generated sections |
| GET | `/<id>/section/<n>` | Get single section content |
| POST | `/chat` | Chat with Report Agent |
| GET | `/<id>/download` | Download report as Markdown |
| GET | `/<id>/agent-log` | Get structured agent execution log |
